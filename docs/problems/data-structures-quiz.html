Data structure basics

Tuples, lists and dictionaries

Last time, we played with our very first “dataset”: the English dictionary text file. This time, we will go a bit further and use a proper dataset from Kaggle, a data science hub that also acts as a repository for cool aggregated data. This quiz is an exploration of what you can do using datasets and Python.

Before you begin, grab the dataset off Kaggle. The file itself is pretty big, and you might need to register for a free Kaggle account to download it. The data itself is stored as a CSV file. CSV files are just like Excel spreadsheets: columns of data are separated by commas (CSV stands for “comma-separated values”) and rows are denoted by lines.

To simplify the task of reading our rows, we can use the csv module built into Python. We can amend our previous file-reading example as follows to use the csv module:

from csv import reader

with open('my_file.csv', 'r') as input_file:
  for row in reader(input_file):
    print(row)

In the scenario above, each row is split into columns, making row a list of string values that you can access using the indexing system discussed in class.

  1. Before we can explore the dataset, we need to make sure that we can access it properly. Write a short script that will open the CSV file, count its rows and print the final count. You can compare your answer to the actual row count, obtained by opening the CSV file with Excel.

  2. The CSV file contains a large number of news headlines from the past few years. Using what you learned in the previous exercises using the dictionary file, can you write a script that will count how many times each word appears in the headlines? For simplicity, you can consider that words that are differently cased (i.e. “News” v. “news”) are the same word. From your result, you should be able to print the ten most common words and how many times they appear in the dataset.

  3. What is the average length of news headlines in our dataset? What about the median?

  4. Design a small search engine that will allow a user to search for headlines using keywords. At first, only consider single keyword searches: can you make your script find headlines containing one or more keywords specified by the user?

The same way we used open() with the r mode to read files, we can use the w mode to create new files and save our content. For example, if we have a list of numbers [1,2,3,4], we can write them to a file using the following snippet:

my_list = [1,2,3,4]
with open('my_file.txt', 'w') as outfile:
  for number in my_list:
    outfile.write(str(number) + '\n')

Note that write() works exactly like print().

  1. Using your newfound ability to write to files, can you build a new CSV that contains one row per unique word in the original data set, each row containing columns that describe the number of occurences of the word and the proportion (between 0 and 1) of headlines in which it appears?

    Hint: to write to a CSV file, you can simply adjust your open() statement and ensure that each line uses a comma (,) as separator between data.
