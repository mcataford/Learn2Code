1. Using the interactive console

For the first part of this lesson, you can use the interactive console through a Command Prompt/Terminal (by opening a Command Prompt/Terminal and typing python) or through IDLE. Since IDLE’s interactive console is just a graphical interface wrapped around the regular Python console, you shouldn’t see any difference in function.

Before we get started, consider a few things about the interactive console:

  1. It has memory. Any variable or function defined in the console will exist and be accessible until the console is terminated. This allows us to type long scripts one statement at a time and keep the results of each statement at hand for the next;
  2. It can be terminated at any time by using the exit statement exit();
  3. If the console is busy with a task, you can force termination using the Control/Command key and C, this is called a keyboard interrupt;
  4. Any statement that produces output will print out its output on the line following a statement;
  5. You can view the contents of a variable at any time by using its name as a statement.

1.1 Mathematical operations

The most basic thing the interactive console can do it basic arithmetic. If you use any mathematical operation as a statement, you should see the results printed below. All the basic operators taught in algebra are valid. As such, we can experiment and produce the following:

Note that just like in classical algebra, you can use parentheses to group operations together, and chained operations such as (1 + 1) * (12 * 50) can be composed from simpler ones.

2. Variables

What differentiates the interactive console from a simple pocket calculator is its ability to store and manipulate different kinds of data. To achieve this, we will use variables. Variables in Python are no different from their algebraic counterparts: they act as a symbolic replacement for data. This means that once defined, a variable can be inserted in other operations and behave as if you had inserted the data is contains.

Before we create our own, however, we should define a few basic data types available in Python: these data types will determine what kind of data is stored in a variable and what sort of operations we can apply on said variable.

2.1 Primitives and Objects

Python data types can be separated into two classes: primitives and objects. The separating criterion is fairly simple: primitives can be represented by a single numerical value while objects require a bigger structure stored in memory, to which the variable points. Below are the three main primitive types we will be using most of the time:

Data type Definition Example values
int Integer values 1, -1, 2000000
float Floating-point values 1.1, -1.5
boolean True or false binary values True, False

In addition to these types, we need to define at least one object type: the string. Strings are collections of characters and can contain single characters, words, sentences and even entire books.

Data type Definition Example values
str Strings, any set of character surrounded by single or double quotation marks "Hello, world!", "A"

2.2 Operations on primitives

All of the arithmetic operations we have experimented with in the first section of this lesson are applicable to any primitive. In the case of True and False (boolean values), those will be interpreted as 1 and 0 respectively. As such, the following is possible:

Arithmetic using boolean values

This outlines what we said earlier: any primitive value can be represented as a number.

2.3 Operations on objects

Operations on objects are a bit trickier. Because objects are complex structures, no two object types have the same operations! Because object operations are defined on a case by case basis, we’ll look at the one object we considered so far: the string.

While strings cannot be divided or subtracted from one another, two operations exist:

Operation Definition
+ Concatenation, or the union of two or more strings
* Multiplication, or the act of repeating strings

The first operation, concatenation, allows you to make longer strings out of shorter ones. This means that given three strings, we can use the + operator to form a new longer one that represents the sum of the three:

Concatenating strings

The same way, using the * operator will duplicate and concatenate strings. For example, typing "hello" * 2 would result in the string "hellohello"; the initial string would be repeated twice, and the repetitions concatenated.

2.4 Operations and data types

The operations defined earlier only apply between items of the same type (either primitive or object). As such, while you can add a float to an int, you cannot concatenate a string to a number directly as either element has a different definition of what the operation + should do. To solve this issue, you can convert one data type to the other, when possible. To do so, we will introduce three functions (also called functions):

Function Definition
str(x) If possible, converts the value x to a string. This would, for example, convert the integer 1 to the string "1".
int(x) If possible, converts the value x to an integer.
float(x) If possible, convers the value x to a float.

These conversions are limited: only strings representing numbers can be converted to proper numbers using float(x) and int(x), but any float, integer or boolean can be converted to a string using str(x).

We can use these functions to combine data of different types. Using this, we could write the following statement:

"Hello! I am " + str(27) + " and I like Python."

Without the type conversion, an error would occur:

TypeError during concatenation

2.5 Creating and modifying variables

As we’ve seen in the previous sections, we can easily manipulate raw data in the interactive console. However, one large issue that stops us from going forward is data persistence. While we can apply operations on numbers and strings, we have yet to save anything to memory! This is where variables come in. As we said earlier, variables are symbolic replacements for data. Once defined, a variable and the data is contains can be substituted in any other statement.

Python being a soft-typed language, we do not need to handle which data type each variable stores; the interpreter will handle that seamlessly as we use our variables. This results in a very simple model for the definition of variables:

my_variable = some_value

Using this model, you can create variables of any of the types we’ve manipulated before:

age = 27
height = 180.1
name = "Marc"

Since variables are symbolic replacements, we can then use those variables by inserting their names, age, height and name where appropriate. We could make use of concatenation with all three to produce a sentence with embedded data:

sentence = "Hello! My name is " + name + " and I am " + str(age) + " years old. I am " + str(height) + " centimetres high."

As we’ve seen before, the calls to str(x) will convert age and height from their respective primitive types to strings, then the + will concatenate all those strings together into Hello! My name is Marc and I am 27 years old. I am 180.1 centimetres high.

3. Writing scripts

To progress further and to start building any sort of project, we will need to be able to execute a multitude of statements at once. To accomplish this, we will write scripts, or sequences of statements that will be executed all at once instead of one-by-one as we did so far in the interactive console.

To write a script, simply save a sequence of statements, one per line, in a file bearing the extension .py. This will be called a Python script file. To execute it all at once, we will not use the interactive console, as it is meant for line-by-line execution. We will turn out attention to IDLE.

Using IDLE, you can create a new script file through the File menu. This opens up a new window in which you can type your script. Just like any other text file, you can save and open scripts using IDLE; this way, you can revisit them later as needed. Once your script file contains statements and is saved, the Run menu will allow you to execute your script file (Run module).

For example, consider the follow sample script: it contains our last experiment with concatenation:

age = 27
height = 180.1
name = "Marc"

sentence = "Hello! My name is " + name + " and I am " + str(age) + " years old. I am " + str(height) + " centimetres high."

print(sentence)

When executing your script, the interpreter will work its way from the top to the bottom and execute your statements one at a time. Since the interpreter is never aware of the whole file, it will only raise an error if it tries to execute a faulty statement.

Notice the use of print(x). Since we are no longer in the interactive console, output is not longer automatically printed to the screen. The print function print(x) will solve that issue by printing the string or variable given to it as parameter.

4. Functions

Functions are meant to be blocks of code you can reuse. Just like functions in mathematics, they can accept input and produce output. Python has dozens of built-in functions, a few of which we’ve seen previously:

Function Definition
str(x) If possible, converts the value x to a string. This would, for example, convert the integer 1 to the string "1".
int(x) If possible, converts the value x to an integer.
float(x) If possible, convers the value x to a float.
print(x) Prints the value of x to the screen.

As we go forward, we will add more functions to this list, but the most important part about Python functions isn’t that the language offers a lot of them out-of-the-box but rather than you can make your own.

The basic model to define a function introduces the def keyword:

def my_function(input):
  #Code that is executed when the function is called.

  return output

A few things to consider:

  1. The def keyword indicates that a function is being defined;
  2. my_function is the given name of our function, we will use it to call our function later (i.e. my_function());
  3. input is a comma-separated list of items that will act as variables within the function;
  4. The return keyword “hands back” the output of the function to whoever called the function. An example of this is str(x), which returns a string made from the value of x.

Reusing our previous example in which we build a string that introduced us, we could write a function that returns our concatenated string and takes the three elements as input:

def introduce_myself(name, age, height):
  output = "Hello! My name is " + name + " and I am " + str(age) +  " years old. I am " + height + " centimetres high."

  return output

Once defined, we can use introduce_myself() anywhere in our script and as long as we give it the right parameters (a string, an integer and a float, in that order), it will produce the right output.

For example, we can use introduce_myself("Marc", 27, 180.1) and get the same output string as before.

4.1 Indentation

You might have noticed that not all lines are on the same level in the previous example. This is not for looks: it’s by design. Blocks of code such as functions use indentation as delimiters in Python, each level being the equivalent of “brackets” that contain the block’s body.

Every time we will declare a block (such as the def statement in the previous example), we will give any following statement that is part of the block one extra level of indentation. As such, we will obtain a layout such as the following:

def my_function1():
  #Code that is in the function "my_function1"

def my_function2():
  #Code that is in the function "my_function2"

#Code that is not in any of the function but that is in the file.
#Any code at this level is executed if you run your script.

Note that any code that is at the “base” level of your script is executed when your script is run.

5. Comments

You might have noticed sentences embedded in code blocks on lines that start with #. These are called comments. Comments are notes left by the programmer to contextualize code, explain the logic of a certain block or provide guidance on things left to do or fix.

You can add comments to your code two ways:


# You can write single-line comments using the hashbang sign before the comment text.

"""
You can also use three consecutive quotation marks to open a block comment
that can span over multiple lines. Another set of three quotation marks will
close the block comment.
"""

Comments are ignored by the interpreter and do not alter the behaviour of your script. You are strongly encouraged to comment your code, as logic that is self-evident today might not remain so through time, and code that you would share might raise questions from other programmers.
