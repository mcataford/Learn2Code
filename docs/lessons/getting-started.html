1. Tools overview

Python doesn’t require much supporting software: the two things we absolutely need are the Python interpreter and a code editor you are comfortable with.

1.1 The Python interpreter and IDLE

This courses uses Python 3.5.x. While you can use previous versions of Python to learn about the language, keep in mind that if you do, syntactic differences will appear along the way that will require you to correct some of the examples. When we will start using third-party modules, the Python version you use will become even more important as different features and module versions are supported by Python 2.7.x and 3.5+. As such, it is strongly recommended to stick with the recommended version.

Regardless of which computer model you have, you can download the Python 3.5+ interpreter here by clicking on the appropriate button.

The Python interpreter download page

The installation package not only contains the interpreter, which will read through our code and execute it, but also IDLE, an easy to use development environment that facilitates writing code and executing it. For the purposes of our course, we will use IDLE as our code editor. If you wish to grow beyond IDLE and use other tools, see Other editors.

No matter the editor you choose to use, once the interpreter is installed, you have access to the interactive Python console. To verify that everything is in working order, you should do the following:

First, open up a command prompt window. If you are running any iteration of Windows, you will find it by looking for Command Prompt, and you can look for Terminal on both OSX and Linux. Once your Command Prompt/Terminal window is open, type in the following:

python

If your interpreter is functional, the interactive console should intialize, showing you which version of Python you are currently using and giving you the opportunity to enter commands.

The interactive Python console

In here, you are able to type any Python statement and see output directly, much like in R or Matlab. We will discuss its usefulness later. To exit the interactive console, simply type exit() and press the return key.

1.2 Git and Github

To keep our code organized, we will use Git. Git is a free and opensource version control system that keeps sharing and keeping track of our projects simple. You can opt to organize your code on your computer as you see fit, but if you choose to use Git, install Git and follow the steps below to create an account and link it to your computer.

We will use Git through the Github, a free service that takes care of maintaining a server on which we can archive our code. To use the service, we will need to create a free account.

1.2.1 Creating an account

Creating a Github account is very simple. First, head to https//www.github.com and fill up the sign up form on the front page. Make sure to use a valid email address as you will have to respond to a validation email to complete the setup.

Github’s signup form

Once you press “Sign up for Github”, you will be taken to the next step: choosing a plan. Github offers paid plans, but for our needs, the free plan is just fine and should be selected by default. If you are currently a student, you most likely quality for the paid version free-of-charge, see the note at the bottom of this page for more details.

Choosing a plan

Once that is done, you should be taken to an optional poll about your programming practice. Feel free to fill it up or to use the “Skip this step” link at the bottom of the form.

Github’s usage poll

Once everything is done, you should receive a confirmation email that will unlock the full potential of your account. Make sure that you click the link that’s on there.

You cannot got further without confirming

1.2.2 Creating repositories

Repositories are containers for your projects. They allow you to group code together based on what you are working on. You can have as many repositories as you desire; today, we will initialize the first one so we can store our course material in it. Once your email is verified, head to your profile by clicking on the “+” located on the top right of the screen and select “New repository”.

Accessing the repository creation form

This will take you to a form defining the basic settings of your repository. Make sure that you do the following:

  1. Choose a meaningful name for your repository (this will be how others and yourself will see it online);
  2. Tick the “Initialize this repository with a README”;
  3. Choose a .gitignore file using the “Add .gitignore” button, you can simply type in Python and use the prebuilt one;

With that done, you can click “Create repository” .

Repository setup

The final step to take on Github’s end is to copy the repository address. This will allow you to create a link between your computer and the repository so you can publish your changes without having to access the website.

Repository address

1.2.3 Setting up Git on your computer to access the repository

Back on Command Prompt or Terminal, you should first type the following to make sure that Git is properly installed:

git --version

If everything is set up correctly, this should display the current version of Git.

You should then navigate to a convenient location to put your Python projects (in this example, we’ll use the My Documents folder):

cd C:\Users\<username>\Documents

Note: On OSX and Linux, this path will be different: /Users/<username>/Documents/ should be equivalent for OSX and /home/<username>/ should be equivalent for Linux

From there, you can clone the repository you created on Github, thereby creating a directory on your computer which is a copy of the Github-hosted version. This is where we’ll store the code we write so we can synchronize the directory with Github’s servers.

git clone <repository address>

Here, <repository address> should have the form https://github.com/<username>/<repository>.git. In the previous examples, the link copied from Github was https://github.com/pythonisbae/firstproject.git.

This will create a directory with the same name as your repository containing your files. You can then access it using cd.

1.2.4 Using Git and Github

At this point, you are ready to use Github! You can test your Git setup by modifying any file in the directory you just created (you can open the README.md file, for example), going back to the Command Prompt/Terminal and using the following commands:

git status

This will show you the current modifications made to the repository. The report should look somewhat like this:

Repository changes

You can then use git add . to add all the files to what you want to synchronize and git commit -m "<message>" to synchronize. Make sure to replace <message> by something meaningful to describe what you changed! This will be useful later when you’ll need to look back at what you’ve done.

Pushing to Github

If all goes well, you should see a synchronization report like the one above. This means that you have successfully uploaded your material to Github. If you go back to your repository’s page on Github’s website, you should now see your updated file(s).
