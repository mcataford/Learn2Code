1. Boolean expressions

In the previous lesson, we mentioned how the boolean datatype was used to represent values that are either true or false. By themselves, these values aren’t very useful: their true purpose is to be the result of boolean expressions that compare values of other types together.

1.1 Comparison operators

The same way that we can use arithmetic operators on numerical values to produce other numerical values (see Mathematical operations), Python includes a number of comparison operators taken from algebraic inequations. The table below is a collection of such operators:

Operator Significance Example
> Greater than 2 > 1
< Lesser than 1 < 2
>= Greater or equal to 5 >= 1
<= Lesser or equal to 1 <= 5
== Equal to 1 == 1
!= Not equal to 1 != 2

As an experiment, you can use the interactive Python console and enter the statements in the Example column of the last table: you should see that all of them evaluate to True. Similarly, you could modify those statements such that they result in False. These are called boolean expressions.

Note that there also exist operators that do not deal with numbers. For example, == can be used between two strings (i.e. "Marc" != "marc") to compare their contents and in can be used to verify if an element is present in a collection (i.e. "a" in "Marc").

Finally, any expression can be negated using not: "d" not in "Marc".

1.2 Composite statements

When we talked about mathematical operations), we mentioned that simple 2-operand mathematical statements can be combined into bigger, richer expressions. We could take 1 + 1 and combine it with 2 * 8 to obtain (1 + 1) + (2 * 8) using parentheses, for example.

This usage of parentheses to combine simple statements into more complex ones also applies to boolean expressions, which can be connected together using and or or.

As their names imply, these operators will allow you to specify whether both or either conditions involved in a combination need to be true for the whole expression to be true. Here are a few examples of and and or in action:

Statement Evaluates to
(1 > 2) or (2 > 0) True
(1 > 2) an (2 > 0) False
((1 > 0) and (2 > 0)) or (1 < 0) True
((1 > 0) and (2 > 0)) and (1 < 0) False

Of course, any of the operators we mentioned before can be used to make up the simple expressions that are combined using or and and.

1.3 Variables in boolean expressions

In the previous slew of examples, we used literal values to make up our expressions. This kind of expression is, evidently, not that interesting since it never changes. When we discussed variables, we mentioned that they were symbolic replacements for data: this means that you can use variables anywhere in boolean expressions - your expressions can now change based on variables that you set.

my_age = input("What is your age")
is_old = int(my_age) > 40

print(is_old)

In the present example, if the variable my_age is a number over 40, we will print the value True, otherwise we will print False. If you are puzzled by input(), take a look at the previous lesson’s quiz, where input(x) was introduced.

2. Conditional statements

Now that we have established that we can use certain operators to obtain boolean values from the data we are manipulating, we can use the outcome of these expressions to guide which statements get executed and which do not. By using boolean expressions as conditions, we can build blocks, called conditional blocks, that will depend on their given condition(s) to be executed.

2.1 if blocks

The simplest conditional blocks are composed of one single unit: the if block. if blocks have a singular mission: to execute the code they contain if and only if the condition that defines them evaluates to True.

Their general form is as follows:

#Any code here is executed before the if block.

if condition:
  #Code that gets executed if 'condition' is True

#Any code here is executed after the if block.

In the event where the condition given to the if block is False, the block is simply skipped. For example, we could selectively print a message based on a number input by the user:

number = int(input("Enter a number: "))

if number > 10:
  print("The number you entered was bigger than 10!")

In the previous snippet, the print() call is only executed if the number number satisfies the if block’s condition, which is number > 10. Any number that doesn’t meet this criterion will cause the condition to be False and the if block to be skipped entirely.

2.2 elif blocks

Sometimes, the if block alone isn’t enough to describe the behaviour that we want to implement. Since if is skipped if its condition is False, multiple if blocks could be used in chain to implement the behaviour “if not the first, check another condition”:

if condition1:
  #Code that is executed if condition1 == True
if not condition1 and condition2:
  #Code that is executed if the first "if" failed but condition2 is True

This form however gets cumbersome very quickly if you have a series of other conditions to check because you always need to ensure that all the previous conditions are False; since the if blocks are independent from each other, a faulty set of conditions could potentially allow them both to be executed one after the other.

To solve this issue and to simplify the implementation of that sort of behaviour, we introduce the elif statement. elif is functionally the same as if, but stands for “else if”. In combination with an if, an elif will only be executed if the if it’s attached to is skipped. We could rewrite the previous block of code as:

if condition1:
  #Code that is executed if condition1 == True
elif condition2:
  #"elif" implies that condition1  was False and checks condition2

This syntax really shines when you consider cases where you have multiple elif blocks:

if condition1:
  #Code that is executed if condition1 == True
elif condition2:
  #"elif" implies that condition1  was False and checks condition2
elif condition3:
  #"elif" implies that condition1 and condition2 were False and checks condition3
elif condition4:
  #"elif" implies that condition1, condition2 and condition3  was False and checks condition4

If we opted to use a contraption of if blocks alone, we would have to ensure that previous conditions did indeed fail before checking if the new condition is True.

2.3 else blocks

THe else blocks complements the if and elif blocks by providing a “catch-all” scenario. Whenever an else is present following a combination of if and elif blocks, it is executed if and only if all the previous conditional blocks fail. This means that all the blocks that have conditions will be skipped and that the else block will be executed before the script can move on.

The else requires no condition by definition: its condition could be expressed as “none of the previous conditions were true”.

if condition1:
  #Code executed if condition1 is true
elif condition2:
  #Code executed if condition2 is true but condition1 is false
else:
  #If all else fails and both conditions are false, this is executed

2.4 A complete model

Combining all three conditional blocks, we can produce a general model for conditional elements. Any valid conditional structure should…

  • …be opened by an if block; Note: Semantically, it wouldn’t make sense to open with an elif or an else since these both depend on a previous element being false.
  • …feature zero or more elif block, as needed;
  • …feature zero or one else block, as needed.

The combination of blocks that will make up your conditional statements will depend on the behaviour you are trying to implement. As such, not all blocks are relevant to all conditional statements: sometimes, a simple if is enough, but in other cases, several elif blocks are needed to get it right.

if condition1:
  """
  Mandatory block for any conditional statement.
  """
elif condition2:
  """
  Optional block, can be present in multiples, as needed.
  """
else:
  """
  Optional block, can only be present once in a statement.
  """

3. A concrete example

In the previous lesson’s problem set, we introduced a temperature conversion problem. If you experimented with different input values for your conversion process, you probably noticed that not all values are valid.

For example, the absolute zero for Celsius degrees is -273.15C, which means that any value below that shouldn’t be converted and should be flagged as invalid. With conditional statements, input given to the conversion function (or collected from the user via input()) can be validated such that only values within a valid range are considered.

We could then add a conditional statement such as this one to ensure the validity of conversion results:

if temperature < -273.15:
  #Return the minimal Fahrenheit value, or an error code.
else:
  #Compute the conversion as usual
