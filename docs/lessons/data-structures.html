1. Data structures

So far, we have been storing data in variables. This allowed us to save partial results, keep data ready for use later and simplify the implementation of otherwise complex operations. The way we have been using variables so far however raised a problem: if we wanted to store a series of data, say 10 observations of a single phenomenon, we would need en equal number of variables. This quickly becomes problematic as the quantity of data we want to store grows: a program with 100s of variables would quickly become unreadable.

To work around this, we can use data structures which permit the common storage of multiple elements under one variable. Using these constructs, we will be able to store more than a single piece of data in each variable, which in turn provides us with a way to collect related data together.

Python defines three main data structures for us: the tuple, the list and the dictionary. We will divide them in two categories: ordered collections (such as tuples and lists, which can be sorted), and unordered collections (which cannot be sorted).

2. Tuples

Tuples are the simplest form of paired data defined by Python. They represent immutable vectors, in other words, groupings of data that cannot be changed once created.

Tuples can store any number of elements of any type, and can be created using the following syntax:

my_tuple = ("Marc",27,180.1)

Note that our tuple is defined using a comma-separated list of element between parentheses. The list in question can be of any length necessary.

Once a tuple is created, individual elements can be accessed using indices.

2.1 Indexing

Indices represent the positions of elements within a collection. In Python, indexing is done using a zero base, which means that the first element of an ordered collection is always 0 and the last one, length - 1. To use indices, we can use the following syntax:

#Prints the first element of the tuple 'my_tuple'
print(my_tuple[0])

Because tuples are immutable, we can access elements as we would any other variable, but we cannot use the assignment operator =. In fact, any statement of the form collection[index] will act exactly as the variables we have seen before.

Since the length of a collection may not always be known (such as in the case where a function receives a tuple as parameter, for example), we can use the len(x) function to find out the length of any collection.

my_tuple = (1,2,3,4,5)

#Prints 5
print(len(my_tuple))

Based on this, the last element of any ordered collection is always len(collection) - 1.

3. Lists

Lists are extremely similar to tuples, with the added benefit of being mutable. Just like tuples, they can contain data of differing types (although this is generally discourages for reasons we will discuss in a later section of this lesson), use a zero-based indexing system and can be accessed using the syntax collection[index]. However, since lists are more complex than tuples, the syntax to create them and to manipulate them will be slightly different:

3.1 Creating lists

We can create lists using the same kind of statement as with tuples:

my_list = [1,2,3,4,5]

Note that we use the square brackets instead of parentheses to encapsulate the comma separated list of values.

We can also create an empty list using list():

my_empty_list = list()

Since we can edit the list later on, creating empty lists might be quite useful depending on what functions we are trying to implement.
