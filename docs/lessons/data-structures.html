1. Data structures

Until now, we have stored single pieces of data in variables. This allowed us to add persistence to our calculations and use the results from previous operations as input for the next. While useful for datum that stands on its own, using variables in the way we have been quickly becomes impractical if we have collections of data that are related in some way.

To illustrate this limit to single-value variables, imagine an experiment where 100 different measurements were collected. Using what we know, storing this data would require the creationg of 100 different variables, producing scripts that quickly become hard to manage, if manageable at all. As we will see later in this lesson, storing these datapoints in separate variables also prevents us from using what we know of loops to iteratively repeat operations on all the data collected.

This is where data structures come in. Python provides us with three primary structures which will allow us to store large amounts of data in an organized fashion that is compatible with loops: tuples, lists and dictionaries. These structures will still be expressed as variables, but will no longer contain single datum. Instead, they will contain a collection of data accessible through indices or keys.

In the discussion that follows, we will call our structures collections, as the next lesson will reveal that we can build our own data structures whose goal isn’t to store large amounts of data at once but to store data in a specific way depending on our needs.

2. Tuples

Tuples are the simplest type of collection available in Python. They represent vectors and are used to create “bundles” of closely-related data that aren’t meant to be edited. For example, we might want to store coordinates, measurement/readings and grouped return values with tuples as none of these are meant to be directly edited; the data they contain is however used in further operations.

2.1 Creating tuples

Creating tuples can be done in a handful of ways:

Using a constructor

Constructors are functions that can generate objects. In the present case, they will generate a default tuple, which is empty. While the read-only nature of tuples makes the constructor useless on its own, its main use will be conversion:

#Creating an empty tuple using the tuple() constructor
my_tuple = tuple()

In the example above, printing the value of my_tuple is (), which represents an empty tuple.

Giving any iterable to our constructor will convert it to a tuple, separating its parts into a list of comma separated elements:

#Splitting a string into a tuple of characters
my_chars = tuple("Hello")

Once converted, my_chars contains ("H", "e", "l", "l", "o"), which is the tuple containing all the iterable elements of the string "Hello".

From this, we can notice a pattern: tuples are comma-separated lists of items surrounded by parentheses.

Using values directly

We can also create tuples by feeding values directly into a variable as a comma-separated list:

numbers = 1, 2, 3, 4, 5
numbers2 = (1, 2, 3, 4, 5)

In the above case, whether you surround your list by parentheses or not, the result will be the same: a tuple containing the values is created and printing either variables will result in (1, 2, 3, 4, 5).

2.2 Accessing tuples

Tuples are classified as ordered collections: elements within tuples have a set position that represents where they are in the structure. We can exploit this property to access single elements.

The position of elements inside of ordered collections like the tuple is represented by the element’s index. Indices are integers ranging from 0 to len(collection) - 1, as such, the first index is 0 and the last is the length of the collection minus 1.

Consider the following tuple, which we created earlier:

numbers = 1, 2, 3, 4, 5, 6

Accessing any of the elements using indices can be done using the model collection[index]. As such, numbers[0] would pull the first element off the numbers tuple, while numbers[5] would pull the last (since the length of our collection is 6, the maximal index is 6 - 1 = 5).

Values pulled from collections can be used just like any other variables would (as long as you do not attempt to edit a value stored in a tuple directly, since tuples are read-only):

#Print the sum of the first two items of the tuple
print(numbers[0]+numbers[1])

#Store a copy of the third value in a variable that you can edit
third = numbers[2]
#Here, you can edit `third` because the variable `third` contains a copy of the read-only value. The copy can be edited.
third = third + 1
print(third)

Be careful

Using indices that are outside of the valid range for a certain collection will cause errors. For instance, in our previous numbers tuple, attempting to access numbers[10] will raise an error and halt the execution of your script because the index you are trying to access is considered unsafe.

2.3 The length of collections

Just like in the case of strings, the length of any collection can be obtained using len(). In this case, calling len(numbers) will return an integer representing the length of the collection numbers.

As such, accessing the last element of a collection, no matter what its length is, can be achieved by using collection[len(collection) - 1] (or access the element of collection that is at index len(collection) - 1).

3. Lists

Lists are another type of ordered collection. Just like tuples, they can contain pieces of data indexed by position. However, contrarily to tuples, lists can be edited – as such, they can grow or shrink depending on your needs.

Because they are not read-only, lists will also possess a certain number of functions that will help you manipulate them.

3.1 Creating lists

Lists can be created using the two standard methods outlined in Creating tuples:

Using a constructor

#Create an empty list that can be expanded later
my_list = list()

#Create a list from another iterable
letters = list("Hello")

Note that we can also convert a tuple to a list to make it editable. We can use the list() constructor to do so by passing a tuple to it as a parameter:

my_tuple = 1, 2, 3
my_list = list(my_tuple)

While my_tuple is read-only, my_list is not and contains the same values as my_tuple. Of course, the opposite can also be achieved: the tuple() constructor can convert a list back into a tuple.

Using values

my_numbers = [1, 2, 3, 4, 5]

Note that we used the square brackets [] instead of parentheses. This is what differentiates the list from the tuple when declaring using values.

3.2 Accessing and editing lists

Using the same indexing scheme as with the tuples, we can access elements from a list:

my_list = [1, 2, 3, 4, 5, 6]

#Prints 1
print(my_list[0])

#Prints 6
print(my_list[len(my_list)-1])

Because our lists are editable, however, we can now use the assignment operator = to replace existing values:

#Replace the first value of the list with 99
my_list[0] = 99

print(my_list)
#[99, 2, 3, 4, 5, 6]

This only allows you to edit existing elements of a list, the next section will cover growing or shrinking a list.

3.3 Growing and shrinking lists

The previous usage of indices only covered indices within the bounds [0, len(collection) - 1]. As such, we cannot use it to append new items to a list or otherwise alter the list in a way that would change its length.

This is where the List functions come into play. Just like you might’ve seen with strings, lists have a certain number of functions that can be used to alter their contents.

Function Effects
append(item) Adds item to the end of the list, growing it.
insert(index, item) Inserts item at index index, provided the index exists in the list.
remove(item) Removes the first occurrence of item in the list
pop(index) Removes the item at index index from the list. If no index is given, removes the last item.

Any of the above functions will alter the list in a way that either grows or shrinks it. The usage model is always the same: collection.function():

my_list = [1, 2, 3, 4, 5]

my_list.append(6) 
#[1, 2, 3, 4, 5, 6]

my_list.insert(1,2)
#[1, 2, 2, 3, 4, 5, 6]

my_list.remove(4)
#[1, 2, 2, 3, 5, 6]

my_list.pop(2)
#[1, 2, 3, 5, 6]

Notice that any of the above functions will change the length of the list. As such, when using lists, it becomes increasingly important to use len(x) when referring to the collection’s length. This way, you can ensure that no matter what the condition of the list is, the maximal index is known.

4. Dictionaries

The third primary collection that we will discuss is the odd one out. While lists and tuples were ordered collections in which elements were ordered by position/indices, dictionaries are best described as unordered collections: the position of elements within the structure is irrelevant, what counts is the keys they are associated with.

You can visualize dictionaries as “bags” of data in which each datum is attached to a unique label. We will refer to the combination of datum and label as key/value pairs.

4.1 Creating dictionaries

Much like the previous two collections, dictionaries can be created using constructors and values:

Using constructors

Following the same pattern as list() and tuple(), the dict() constructor can be used to create empty dictionaries or convert pre-existing data structures into dictionaries.

#Creates an empty dictionary
my_dict = dict()

Conversion is not as straightforward as with lists and tuples because instead of having an ordered sequence of items, we now have key/value pairs, each of which is represented by two elements of data.

We can use the dict() constructor to build a dictionary from a list of tuples in which tuples represent key/value pairs:

#A list of key/value pairs
key_values = [("first", 1), ("second", 2)]
my_dict = dict(key_values)

Using values

You can also initialize a dictionary by using a sequence of key/value pairs formatted as follows:

my_dict = {"first": 1, "second": 2}

Note the use of {} braces. Much like tuples and parentheses, and lists and square brackets, the braces are associated with dictionaries. Inside those braces is a comma-separated list of pairs of the form key: value.

4.2 Accessing and editing dictionaries

As we mentioned before, dictionaries do not have indices because they are not ordered. However, we can substitute indices for keys and use the same collection[key] format to access and edit entries:

foods = {"Orange": "Fruit", "Tomato": "Who knows.", "Hot dog": "Definitely not a fruit"}

#Prints "Fruit"
print(foods["Orange"])

#Prints "Definitely not a fruit"
print(foods["Hot dog"])

#Changes the value associated with the key "Orange"
foods["Orange"] = "A yummy snack"

Moreover, because we are dealing with keys instead of indices, insertion can be done by specifying a key that isn’t already in the dictionary and assigning a value to it:

foods["Hamburger"] = "Patties on a bun"

In this case, the foods dictionary is grown by one key/value pair.

To achieve the opposite, you can use the del keyword:

del foods["Orange"]

Once the statement above is executed, the key "Orange" no longer is in the dictionary foods. This also removes the value associated with the key.

Be careful

Accessing a dictionary using a key that doesn’t exist but without assigning a new value will cause an error, much like accessing an index that is out of bounds. To avoid this problem, you can use the get(x) function mentioned in the following section.

4.3 Dictionary functions

Much like lists, dictionaries have a handful of functions that you can use on them:

Function Effect
get(x) Returns the value associated with the key x or None is x doesn’t exist.
values() Returns a list of all the values present in the dictionary.
keys() Returns a list of all the keys present in the dictionary.
items() Returns a list of tuples representing all the key/value pairs.
update(x) Merges the current dictionary with the dictionary x

You can use these functions the same way you would use list functions:

foods = {"Orange": "Fruit", "Tomato": "Who knows.", "Hot dog": "Definitely not a fruit"}

drinks = {"Pop": "Fizzy drinks", "Orange juice": "Almost a fruit"}

#Prints a list of values contained in the foods dictionary
print(list(foods.values()))

#Prints a list of all the keys contained in the drinks dictionary
print(list(drinks.keys()))

#Merges the dictionaries, foods now contains all of drinks.
foods.update(drinks)

5. Collections and loops

Here and there, we mentioned the term enumerable when describing collections and strings. The term itself neans that the value on which it is applied can be looped over and processed piece by piece.

In the context of strings, using a loop would allow us to get individual characters, as we have seen earlier:

for character in "Hello, world!":
  print(character)

In the case of ordered collections, the same structure would simply loop over the items in-order:

numbers = [1, 2, 3, 4, 5]

for num in numbers:
  print(num)

In this case, num will take the value of each item in numbers sequentially.

We can also use our range() function to generate indices. Remember that range() will generate a range of numbers between 0 (unless set to another minimum) and the given maximum. As such, the valid indices in numbers can be described as range(0, len(numbers)) since the maximum is exclusive:

for index in range(0, len(numbers)):
  print(numbers[index])

This last snippet will have the same effect as before: each element of the list will be printed sequentially. The only difference is the approach: in the first, we prioritize the value and discard the index whereas in the second, we prioritize the position and then use each index to get the value.

This model differs sensibly when we deal with dictionaries, as we cannot generate a range of keys. Instead, we will loop through the available keys and use each key to access values:

foods = {"Orange": "Fruit", "Tomato": "Who knows.", "Hot dog": "Definitely not a fruit"}

for food in foods:
  print(food, foods[food])

Here, we go through all the available keys (for food in foods), and at each step, we print both the key (food) and the value (foods[food]).

Be careful

You should always be wary of altering collections while iterating over them. For example, a loop in which you append items to a list might never end if your frame of reference is “until the end of the list”, and one in which you alter the indices might not work very well if you rely on indices to access data.
